# Ivaronix Build Focus

## Core Rule

Build the core trust workflow first.

```text
Private docs/code review
-> consensus
-> burn mode
-> action receipt
-> proof explorer
-> 0G Storage + 0G Chain proof
-> agent passport
-> CLI verify
```

## Added Strategic Upgrades

These are now part of the best Ivaronix direction:

- Proof Explorer
- Memory Permission Center
- Team Workspace First-Class
- Agent Safety Guard
- Forge CLI Proof Mode
- Receipt-Based Reputation
- Operator Pays Mode

## First Build Includes

- basic Proof Explorer receipt page
- Memory Permission Center for agent/skill/session access
- Agent Safety Guard before risky actions
- Forge CLI proof mode
- receipt-based reputation counters
- Operator Pays Mode for low-friction testnet onboarding

## Build Soon After MVP

- team workspace alpha
- shared team memory
- team audit logs
- approval workflows

## Do Not Build First

- Full marketplace
- Payments
- Full enterprise team workspace
- 100 skills
- Mobile app
- Browser extension
- Agent-to-agent economy
- NFT marketplace
- Full enterprise dashboard

## Why

The first version must feel sharp and fully working. A focused 0G-native trust workflow will beat a broad platform with many half-built features.
