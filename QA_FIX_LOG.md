Pair with `QA_TEST_PROGRESS.md` (test status tracker) and `docs/USER_TODO.md` (truly external blockers).

## Header

```
# QA Fix Log · ivaronix.vercel.app · commit <sha>
Open issues:    XX
Fixed:          XX
Pending:        XX
Blocked:        XX
Last updated:   YYYY-MM-DD HH:MM
```

## Per-Row Fields (one row per bug/fix)

Every fix entry must include all 10 fields below. Skipping any field means the fix is not fully documented.

| Field | Meaning |
|---|---|
| Issue | What was broken. |
| Expected behavior | What should happen. |
| Root cause | Why it broke. |
| Fix plan | How it will be fixed. |
| Files changed | Exact files touched. |
| Tests run | Commands and user flows rerun. |
| Evidence | Proof that the fix works (screenshot/video/receipt/tx/CLI output path inside `QA_PROOF_PACK/`). |
| Regression check | Nearby flows checked after fix. |
| Remaining risk | Anything still weak or pending. |
| Status | `PASS`, `FAIL`, `PENDING`, or `BLOCKED`. |
