## Operating rules distilled (do NOT replace the verbatim above)

1. **Stop condition:** every shipped feature has a status of `verified end-to-end with proof`, `fixed and re-tested`, OR `explicitly blocked with a real reason and unblock action`.
2. **No partial credits:** connect-only / screenshots-only / CLI-only / web-only DO NOT count as done.
3. **Test like a user, prove every feature once cleanly** — UI surface AND CLI surface AND smooth + polished.
4. **No compromise. No monkey moves.** Use wisdom, not loopholes.
5. **PMF filter on every additive change**: is it proven? do users care? would a showcase project ship it? If not, don't add it.
6. **Brand contract**: `brand/Ivaronix.html` is canonical. Studio must look indistinguishable side-by-side. Voice = receipts > rhetoric (CLAUDE.md §9). Tokens: paper #FAFAF7, ink #0A0A0A, ink-soft #111111, header sticky 64px backdrop-blur(20px).
7. **Receipts are the spine**. Every action ships one. Schema: zod-validated, canonical-hash-bound, AgentPassport-resolvable signer, ReceiptRegistry-anchored, `ivaronix receipt verify --tee-independent` re-verifiable.
8. **Tier marking discipline**: TIER 1 TEE = router/0G compute (green). TIER 2 EXTERNAL = NVIDIA NIM / OpenAI / Ollama (amber). Honest > flattering.
9. **Track focus**: Track 1 (cognitive backbone for OpenClaw) primary, Track 3 (Agent-as-a-Service via SkillRegistry + creator/treasury fee split) automatic secondary. Skip Tracks 2 + 4.
10. **Before stopping, think 3 times**. If anything is testable that hasn't been tested, do it.
11. **Evidence folder rule**: every screenshot, video, transaction hash, receipt URL, proof URL, and command output used as proof must be linked or named in `QA_TEST_PROGRESS.md`. If proof exists only in a screenshots/videos folder but the progress file does not point to it, the test is not fully recorded.
12. **Mainnet gate**: before saying Ivaronix is completely ready for mainnet, run one explicit final checklist covering contracts, env vars, deployer wallet, chain IDs, RPCs, 0G Compute key, 0G Storage path, receipt anchoring, Proof Explorer, passport state, memory grant/revoke, burn-mode receipt, and one full fresh user flow. Only after that can mainnet readiness be claimed.
13. **Competitor review cadence**: after QA is green, compare once against `new-entries`, `entries`, and `og-projects-showcase`. Do not re-read competitors every 2 minutes. Use the comparison to decide what is still strategically missing, then return to build/test.
14. **Feature add rule**: only add a feature if it closes a judging gap or a proven competitor/PMF gap. Do not add features because they sound cool. Every new feature must improve 0G depth, technical completeness, product value, UX/demo quality, or documentation/submission strength.
